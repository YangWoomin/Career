
## 간단한 인사말

안녕하세요. 기술과 경험에 항상 목말라있는 개발자 양우민 입니다.

이 인사말을 쓰면서 저를 형용하는 "기술과 경험에 항상 목말라있는 개발자" 라는 문구를 붙이면서 잠깐 고민을 했습니다.

나를 표현하기에 이보다 더 좋은 문구는 없을 것 같은데 5년 뒤, 10년 뒤, 20년 뒤에도 저 문구가 계속 유효할까? 라는 생각이 들었습니다.

자신의 총 경력이나 포트폴리오를 소개하는 메인 페이지에 적는 인사말은 자신의 변하지 않는 강력한 장점을 잘 나타내는게 중요하다고 생각하기 때문입니다.

이렇게 짧은 고민을 했지만, 지금까지 살아온 인생을 뒤돌아 봤을 때 앞으로도 바뀌지 않을 것이라는 확신이 들었습니다.

이유는 제가 근본적으로 욕심이 많아서 입니다.

공동체에서 없어서는 안 될 구성원으로서 살아가는 것이 제 인생의 가장 큰 낙인데,

없어서는 안 될 구성원이 되려면 계속 자신을 채찍질하며 성장시켜야 하고 이것이 본능적으로 자기 자신을 채우려는 욕심으로 나타나는 것 같습니다.

얼마나 욕심이 많길래? 라고 생각들지 않으신가요?

아래에 제가 지금까지 해 온 것들을 정리해두었습니다.

C/C++ 서버 개발부터 웹 프론트엔드, 툴, CI/CD, 인프라... 회사에서 하던 것들 외에 자기 계발까지.

제가 지금까지 해 온 것들과 앞으로의 성장 가능성을 확인하실 수 있으실 것입니다.

> 게임 컨텐츠 개발을 제외한 나머지 일들을 주로 해오다 보니 서버의 기반 작업/외부 연동이나 오픈소스, 인프라 중심의 많은 것들을 경험해 보았습니다. 인증 서버에서 사내 계정으로 게임 로그인을 지원하기 위한 nano 인증 연동, 스팀 계정으로 플레이하기 위한 스팀 API 연동 등을 작업했으며, 로비 서버의 scale out이 가능한 형태로의 개선을 위한 유저 세션 저장소인 redis 연동, 서버간 연결 복잡도를 낮추고 메시지 송수신 구현의 의존성을 낮추기 위한 메시지 큐(kafka) 연동 등이 최근에 수행한 작업입니다. 그 외 Make/CMake를 사용한 서버 빌드, 팀시티를 사용한 서버 CI 등을 지원하였습니다. 인프라 쪽은 서버 배포 및 실행을 위한 docker, k8s, 그리고 ansible 등을 사용했으며 서버의 지표와 로그 수집을 위한 InfluxDB + Grafana 그리고 Elasticsearch + Kibana + Fluentd 등을 운영하였습니다.
> 
> 주 언어는 아무래도 지금까지 작업해온 서버들의 구현 언어인 C/C++이며 modern C++도 많은 관심을 가지고 있습니다. Golang이나 C#, Python 등의 언어도 서버 프로토타입 개발이나 여러 툴에도 사용해본 경험이 있어서 다양한 언어들을 사용하는 것을 선호합니다. 해보고 싶은 일들은 지금껏 해온 일들과 비슷한 플랫폼 연동 개발이나 기반 시스템 개발에 관심이 있으며 컨텐츠도 해보고 싶은 생각이 있습니다. 특히 기반 시스템 개발에 관심이 있어서 이번에 준비한 포트폴리오를 지금까지 일하며 습득한 내용을 기반으로 준비를 했습니다.
